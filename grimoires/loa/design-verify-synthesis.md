# Design-Verify Synthesis — @hosaka/crate SDD

> Provenance: adversarial design-verify workflow `wf_176bccea-8a7` (7 dimensions × design + 3 adversarial lenses + synthesis = 29 agents, ~1.03M tokens, 164 tool calls). Verify phase re-run after a script fix. Every claim was re-grounded against the live `openapi.json` by the agents (jq-verified).
> Generated 2026-06-26. This is the locked design input for `sdd.md`.

## Locked Design Decisions (per dimension)
### client-surface
**Decision:** Crate class with constructor opts {apiKey?, baseUrl?, fetch?, timeout?, maxRetries?, headers?} and trailing per-call RequestOptions. /api/v1 is an internal const; baseUrl is ORIGIN-ONLY (validate it has no path and throw if it does, since new URL('/api/v1'+path, base) silently drops a base path). Flat conveniences resolve()/artist()/bandcamp plus dossier/wayfind namespaces. resolve() takes a discriminated-union ResolveQuery (compile-time exactly-one) + runtime guard; discogs accepts string|number coerced via String(). artist(key): HEX64 -> direct, slug -> direct (one hop), locator (discogs:/mbid:) -> resolve-then-direct. UNIFORM gap behavior: locator-miss does NOT throw by default; drop the two-overload onGap and provide a SEPARATE artistOrNull(key): Promise<ArtistDossierContract|null> so inference is stable under dynamically-built options. bandcamp is a callable function-object (call sig + .bulk + .bulkAll), no Proxy. master(id), masters(ids), usage() are on the surface but REQUIRE apiKey: throw a client-side CrateError{kind:'validation'} when called without apiKey, and fix all DX examples so anonymous clients never call them. Beacons (search-events/*) stay cut, rationale corrected to BeaconBearerAuth (per-search JWT), not X-API-Key/401. Type re-exports via narrow aliases off components['schemas'] for ALL public-method return types, not just 5. SearchParams cube_quadrant/exclude_artist/exclude_label typed string|string[] per spec anyOf.

**Why:** MEDIUM = 1:1 typed methods + the two named conveniences + cursor iterator, matching stripe-node/supabase. Per-op auth and the baseUrl join are real spec/runtime correctness issues the original design got wrong.

**Changed after adversarial attack:** master/masters/usage now require apiKey (spec declares ApiKeyAuth + 401/402 on them, verified; the blanket 'security:null' briefing was wrong); onGap two-overload replaced with separate artistOrNull() (TS2769 on dynamic options); locator gap behavior made uniform (no throw-by-default); baseUrl restricted to origin-only with validation; SearchParams faceted params widened to string|string[]; beacon rationale corrected to BeaconBearerAuth; type re-exports expanded to full method surface.

### retry-policy
**Decision:** Pure src/retry.ts (isRetryable, computeDelay, parseRetryAfter) + src/http.ts owning the loop and timers. Retryable status SUPERSET = {429,500,503,504} applied uniformly, stated as a safe over-approximation (per-op declarations are subsets), NOT 'every op declares exactly these'. Full-jitter exponential backoff. Idempotency via explicit per-request boolean set at the method layer (GETs + read-shaped POSTs masters/batch + wayfind/* = true; beacons = false). NEVER retry 400/401/402/404/413/AbortError. 402 reason corrected to 'payment required (past_due/suspended key)'. Retry-After precedence: header (delta-seconds OR HTTP-date) > body retry_after_seconds > jittered backoff; clamp to maxRetryAfterMs AND remaining deadline; validate non-finite/negative as undefined (fall through to backoff). Rename maxAttempts -> maxRetries. Rename caps to maxBackoffMs (jitter cap) and maxRetryAfterMs (server-directed clamp). totalDeadlineMs DEFAULTS finite (120000) with explicit null opt-out. Per-attempt: fresh AbortController + setTimeout cleared in finally (NOT uncancelable AbortSignal.timeout); compose with user signal; disambiguate timeout vs user-abort via signal.reason, not a shared aborted flag. Wrap response.json() in try/catch -> CrateError{kind:'parse'} on BOTH 2xx and error bodies. For header/body-derived delays apply jitter on top of the server value to break thundering-herd.

**Why:** Full jitter + Retry-After-honoring + idempotency-by-flag is correct MEDIUM. The grounding claim and naming were the defects.

**Changed after adversarial attack:** 'declares exactly these' reworded to safe-superset (verified: wayfind/interpret={429,503}, dossier/manifest={429,500}, usage no 503/504); maxAttempts renamed maxRetries; delay caps renamed self-documenting; totalDeadlineMs finite-by-default (unbounded ~3min block was the headline footgun); per-attempt timer uses cancelable setTimeout+AbortController; body retry_after validated negative/NaN; 402 reason corrected; JSON-parse path defined; abort-cause race fixed via signal.reason.

### pagination
**Decision:** ONE dual-iterable handle from bulk(): default for-await yields rows, .pages() yields pages, both share one page generator. source is OPTIONAL (omitted -> send no source param, let server apply documented signals_mbid default; do NOT force a magic string). Loop driven SOLELY by next_cursor, terminate on null. ADD forward-progress guard: if next_cursor equals the cursor just used (or a seen cursor), throw CrateError{kind:'pagination'}; optional maxPages cap. Runtime-validate page shape before iterating: if !Array.isArray(page.rows) or next_cursor neither string nor null, throw CrateError (malformed response). Move the limit guard INTO the generator so ALL errors surface at the same .next() site. limit sent as STRING query param (spec is bare type:string); the 1..200 bound is DESCRIPTION-ONLY folklore (not schema) so keep the client-side clamp but document it as such. Validate source non-empty if provided. opts.cursor typed string|null. Drop throw-in-finally on abort (silent cleanup); check signal.aborted at top of generator before first fetch. Export BandcampRow = BandcampBulkPage['rows'][number]; generic <T=Record<string,unknown>>. Note the no-param MANIFEST branch is out of scope and mistyped as BandcampBulkPage in the spec (flag to API owner).

**Why:** Async-generator gives clean error/abort semantics; next_cursor-only matches the spec (_links absent from BandcampBulkPage). The infinite-loop and malformed-200 gaps were genuine hazards.

**Changed after adversarial attack:** source made optional to honor the documented default; forward-progress guard added (unguarded infinite loop was the top failure mode); runtime page-shape validation added (status-200 garbage leaked a raw TypeError); limit guard moved into generator (two throw sites broke try/catch); throw-in-finally removed; cursor typed string|null; limit 1..200 documented as description-only; BandcampRow exported.

### error-model
**Decision:** Three-class hierarchy rooted at CrateError extends Error: CrateAPIError (non-2xx, defensively-parsed open-schema body + teaching fields read only when valid type, retryAfter, requestId best-effort, masterId, raw body escape hatch, retryable from {429,500,503,504}), CrateNetworkError, CrateTimeoutError, CrateAbortError (distinct from timeout). HARDCODE this.name = string literal per subclass (NOT new.target.name; minification mangles it). Call super(message,{cause}) natively (typed, non-enumerable); drop (this as any).cause. ADD Symbol.for('hosaka.crate.error') brand on the prototype and make all isCrate* guards BRAND-based (not raw instanceof) to survive the dual ESM+CJS boundary; document prefer-guards-over-instanceof. EXPORT symmetric guards isCrateNetworkError/isCrateTimeoutError/isCrateAbortError plus isCrateError/isCrateAPIError/isRateLimited/isRetryable. composeMessage must NOT trust Response.statusText (empty over HTTP/2 + Cloudflare): use a status->reason-phrase table and collapse double spaces; deterministic fallback when body has only {error}. Cap preserved body.raw (~2KB). Body via response.text()+JSON.parse (single read) wrapped so parse failure yields a usable CrateAPIError. classifyTransportError({timedOut,callerAborted,timeoutMs}).

**Why:** Status-as-data + small class set is correct MEDIUM. The minification, dual-package-instanceof, and HTTP/2-statusText defects degrade silently in exactly this SDK's target environments.

**Changed after adversarial attack:** this.name hardcoded per class; cause via native super() not as-any; brand-based guards added for the dual ESM+CJS instanceof hazard (headline build-coupled flaw); symmetric transport-error guards exported; statusText replaced with reason-phrase table (HTTP/2 empties it); body.raw size-capped; deterministic message fallback for teaching-field-less bodies.

### types-and-drift
**Decision:** Vendored spec/openapi.json (byte-pinned) + committed src/generated/crate-api.d.ts (openapi-typescript@7.13.0 EXACT-pinned) + two-tier guard. Tier 1: OFFLINE byte-equality drift test (regenerate from vendored spec via the SAME pinned binary the gen script uses; assert byte-equal) GATES PR CI. Tier 2: network live-spec staleness check on SCHEDULE only (opens an issue on drift). .gitattributes LF on *.d.ts and openapi.json. UNIFY hashing: store sha256(JCS(openapi.json)) in meta.json to match the staleness comparator (use JCS in both). src/types.ts re-exports narrow aliases off components['schemas'] for the FULL method return surface (IdentityResolution, ArtistDossierContract, ApiRootIndex, BandcampBulkPage, BandcampFeedContract, SearchResponse, BreakoutsResponse, FacetCounts, TastemakersResponse, OnesToWatchResponse, MasterEnrichment, BatchResponse, MasterDossierContract, FestivalDossierContract, LabelDossierContract, DossierManifest, UsageResponse, WayfindAnswerResponse, WayfindInterpretResponse) + BandcampRow. Contract test (vitest --typecheck): STRENGTHEN to toEqualTypeOf for nullable scalars (cluster_id/slug/display string|null, locators.discogs number|null, resolved_via, resolved_from, next_cursor string|null); .not.toBeAny() on all aliases; positive assertion that --typecheck engaged (guarded @ts-expect-error) so the guard can't silently no-op; test that fails if any public-method return schema lacks an exported alias.

**Why:** Regenerate-and-byte-compare catches stale-spec + hand-edit + tool-drift in one assertion; offline gate stays deterministic, network check exiled to schedule.

**Changed after adversarial attack:** hashing regime unified (meta.json raw-bytes vs staleness JCS was inconsistent); type re-export surface expanded to ALL return types (5-of-18 left half the SDK unnameable); nullable assertions strengthened toHaveProperty->toEqualTypeOf; --typecheck-engaged positive assertion added; alias-coverage test added.

### build-packaging
**Decision:** Build tool: tsup (esbuild-based, devDep only, zero runtime deps). Dual ESM+CJS via format:['esm','cjs'], dts:true emitting index.d.ts + index.d.cts wired per-condition (arethetypeswrong-clean). package.json: types-first then default in each condition; import/require before fallbacks; engines node>=18; sideEffects:false; files [dist,README,LICENSE]. LICENSE: NOT MIT; spec declares 'Commercial — see Terms of Service'. Set license to UNLICENSED (or the commercial identifier the API owner confirms) and ship a matching LICENSE; do NOT publish the official client of a paid commercial API as MIT. PIN openapi-typescript EXACTLY (7.13.0, not ^7) and typescript tightly, commit lockfile. Pin @arethetypeswrong/cli exactly. Publish gated OFF via private:true PLUS release.yml job guarded by PUBLISH_ENABLED var + NPM_TOKEN-absent no-op. Use prepack (not prepublishOnly) so npm pack packs fresh dist. CI: npm ci -> tsc --noEmit on hand-written src (esbuild strips types unchecked) -> drift test -> tsup build -> arethetypeswrong --pack -> pack tarball, npm-install into throwaway fixture, import(ESM)+require(CJS) the bare specifier from there -> assert dependencies empty -> npm pack --dry-run asserts tarball excludes src/tests/openapi.json. Drift test: generate to temp file and git diff --no-index --exit-code committed temp (avoids stdin newline parity) with an actionable failure message. Add a runtime fetch-presence check in the constructor throwing a teaching CrateError on Node<18.

**Why:** tsup gives arethetypeswrong-clean dual .d.ts with a 12-line config and no convention lock-in; private:true is the fail-closed publish gate.

**Changed after adversarial attack:** license corrected MIT->commercial/UNLICENSED (spec is Commercial, verified — a shippable legal error); openapi-typescript pinned exactly (caret defeats the drift gate); standalone tsc --noEmit added (esbuild strips types unchecked — worst DX failure for a typed SDK); prepack replaces prepublishOnly; smoke test installs the packed tarball into a fixture; pack-contents assertion added; CrateError brand/dual-package note coordinated with error-model.

### testing
**Decision:** Runner: vitest. Mock layer: undici MockAgent via setGlobalDispatcher (disableNetConnect + assertNoPendingInterceptors). Fake timers: vi.advanceTimersByTimeAsync (SDK timeout via setTimeout+AbortController, NOT AbortSignal.timeout). Fixtures GROUNDED in schema: locators.discogs number|null (not string), resolved_from present non-null, resolved_via in {discogs,cluster,null}; validate fixtures against the schema in a self-test. Intercepts assert FULL /api/v1/... paths. Drop '404' from the resolve retry test (resolve declares no 404). Pin query-serialization in a dedicated test (URLSearchParams '+' vs encodeURIComponent '%20', numeric->string). ADD coverage: Retry-After HTTP-date form; dual-package test importing built dist (require(../dist/index.cjs) AND import(../dist/index.js)) asserting CrateError brand/guard cross-format; AbortListener cleanup across retries; concurrent-call independent retry state; partial-page-failure (page2->500) iterator throws cleanly; non-advancing-cursor guard; 200-with-invalid-JSON; consumer AbortSignal composition with internal timeout; no-source bulk; bandcamp/{artistKey} discogs:/mbid: key forms + unresolved-200-sparse; master/masters/usage without apiKey -> client-side CrateError. Live smoke gated CRATE_LIVE_SMOKE=1, single anonymous call. Backoff math unit-tested as pure fn.

**Why:** MockAgent is the native highest-fidelity interception for Node global fetch; vitest fake timers drive backoff deterministically. Added cases cover the named failure modes the original suite missed.

**Changed after adversarial attack:** fixtures corrected to schema (discogs number not string); 404 dropped from resolve retry test; query-serialization pinned (every intercept depends on it); added dual-package/dist, Retry-After HTTP-date, AbortListener-leak, concurrency, partial-page, non-advancing-cursor, 200-invalid-JSON, consumer-signal-composition, no-source-bulk, bandcamp feed key-forms, and auth-required-without-key cases; timeout test pinned to setTimeout+AbortController.

## Ranked Risks
- **[critical]** LICENSE: publishing the official client of a commercially-licensed paid API (spec info.license = 'Commercial — see Terms of Service') under MIT is a legal/contract misrepresentation. Verified in spec.
  - _Mitigation:_ Set package.json license to UNLICENSED (or the specific commercial identifier the API owner confirms) and ship a matching commercial LICENSE; escalate the exact string to the API owner before the publish gate is flipped. Until confirmed, keep private:true.
- **[high]** master(), masters(), usage() hard-require ApiKeyAuth (spec declares ApiKeyAuth + 401/402), but the briefing's blanket 'security:null today' is false for these three. An anonymous client calling them 401/402s at runtime.
  - _Mitigation:_ Throw a client-side CrateError{kind:'validation'} when these are called without apiKey; document the requirement; remove these calls from every anonymous DX example. resolve/artist/bandcamp/search/dossier/wayfind remain genuinely anonymous (security:null verified).
- **[high]** bulk pagination infinite loop: termination is next_cursor===null only; a non-advancing/cycling cursor or rows:[]+non-null-cursor spins forever, amplified by per-page retry.
  - _Mitigation:_ Forward-progress guard (throw if next_cursor equals the cursor just used or a seen cursor) + optional maxPages cap, in the single page generator.
- **[high]** Dual ESM+CJS instanceof hazard: the error model and the retry loop's `e instanceof CrateError` short-circuit break if ESM and CJS each carry their own CrateError class. The LOCKED dual build is exactly the trigger.
  - _Mitigation:_ Symbol.for brand on the prototype; brand-based isCrate* guards; document prefer-guards-over-instanceof; dual-package test importing both built artifacts.
- **[high]** this.name = new.target.name and unchecked types: minification mangles class names (garbage .name/stack traces) and tsup/esbuild strips types without typechecking the hand-written public surface, so a type error in the typed SDK's own signatures ships silently.
  - _Mitigation:_ Hardcode this.name string literal per subclass; add a dedicated tsc --noEmit CI gate over src/ before the tsup build.
- **[medium]** Drift gate phantom failures: caret-ranged openapi-typescript/typescript change generated bytes on an unrelated PR; trailing-newline/banner parity via git diff stdin; CRLF on Windows checkouts.
  - _Mitigation:_ Exact-pin openapi-typescript (7.13.0) + commit lockfile + npm ci; generate to temp file and git diff --no-index --exit-code committed temp; .gitattributes LF on *.d.ts and openapi.json.
- **[medium]** Timeout vs caller-abort and timer leaks: AbortSignal.timeout is uncancelable (leaks 30s timers under pagination/concurrency); same DOMException for both abort causes risks misclassifying a retryable timeout as terminal cancellation; AbortListeners accumulate on a long-lived user signal across retries.
  - _Mitigation:_ Per-attempt fresh AbortController + setTimeout cleared in finally; disambiguate via signal.reason; remove listeners on settle; transport-level test asserting bounded listener count and signal propagation.
- **[medium]** Retry-After / status assumptions: header may be HTTP-date; body retry_after_seconds may be fractional/negative/NaN; statusText is empty over HTTP/2+Cloudflare degrading every error message; non-JSON 2xx/5xx bodies throw on .json().
  - _Mitigation:_ Parse both Retry-After forms; validate non-finite/negative as undefined->backoff; status->reason-phrase table; wrap text()+JSON.parse so parse failures yield CrateError{kind:'parse'} and cap body.raw size.
- **[medium]** Under-typed/under-exported surface: SearchParams dropped cube_quadrant/exclude_artist/exclude_label array forms (spec anyOf string|array, verified); only 5 of 18 return types re-exported; bandcamp limit 1..200 is description-only folklore, not schema.
  - _Mitigation:_ Type the three faceted params string|string[]; re-export all method return types as narrow aliases + a coverage test; keep the limit clamp but document it as description-sourced.

## Recommended Approaches (one pick each)
- **build_tool:** tsup (esbuild-based, devDependency, zero runtime deps; format:['esm','cjs'] + dts:true emits index.d.ts + index.d.cts wired per exports condition, arethetypeswrong-clean). openapi-typescript pinned EXACTLY at 7.13.0.
- **test_runner:** vitest
- **http_mock_layer:** undici MockAgent via setGlobalDispatcher (intercepts the exact undici-backed Node global fetch; disableNetConnect + assertNoPendingInterceptors; replyWithError + real AbortController for socket/abort cases). NOT msw/nock/fetch-stub.
- **fake_timers:** vitest vi.useFakeTimers + vi.advanceTimersByTimeAsync; SDK timeout MUST be setTimeout+AbortController (not AbortSignal.timeout) so fake timers control it and the timer is cancelable.
- **drift_test_mechanism:** Two-tier: (1) OFFLINE byte-equality — regenerate from vendored spec/openapi.json via the SAME pinned openapi-typescript binary the gen script uses, to a temp file, git diff --no-index --exit-code committed temp, GATES PR CI; (2) NETWORK live-spec staleness on a schedule only, opens an issue, never gates PRs. .gitattributes LF + committed lockfile for reproducibility.
- **error_dual_package_strategy:** Symbol.for('hosaka.crate.error') brand on prototypes + brand-based guard functions; hardcoded per-class this.name string literals; native super(message,{cause}). Document prefer-guards-over-instanceof.
- **publish_gate:** private:true (fail-closed) + release.yml job guarded by PUBLISH_ENABLED repo var and NPM_TOKEN-absent no-op; prepack runs the build; unlock = set var + add secret + flip private in the version-bump commit.
- **retry_config_naming:** maxRetries (count of retries), maxBackoffMs (jitter cap), maxRetryAfterMs (server-directed clamp), totalDeadlineMs (finite default 120000, null to opt out), perAttemptTimeoutMs.
- **pagination_shape:** Single dual-iterable handle: default for-await yields rows, .pages() yields pages, shared page generator; source optional; forward-progress + page-shape guards; export BandcampRow alias.
