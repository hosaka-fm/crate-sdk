# Agent Ergonomics Requirements — @hosaka/crate (ADX-1..10)

> Provenance: agent-ergonomics workflow `wf_7af47989-2b7` — adapted the imported agent-ergonomics 11-dimension rubric + the MCP/agent-API-design lens to a TypeScript library SDK whose primary consumer is AI agents, scored the SDD surface, synthesized to 10 prioritized requirements. Folded into sdd.md §15 + sprint.md S2/S3. Generated 2026-06-26.

**Verdict:** The SDD is already strong on the structural agent-DX axioms — never-silent-fail (§7 all-throw + honest-gap passthrough), a stable .kind discriminant, brand-based dual-package guards, and honest provenance surfaced verbatim — but it has material, dedupe-able gaps at exactly the agent's success-and-recovery moments: error payloads aren't serializable (toJSON), the kind/code taxonomy and SDK self-description aren't exported, client-side errors don't author the fix they know, the agent's primary docs (JSDoc @example + a README agent recipe) aren't required, and the most natural first call crate.resolve('Four Tet') is a hard error. Ten tight requirements (4 must, 6 should), mapped to S2-errors / S3-surface / S3-docs / S3-tests, close them without any agent-mode bloat.

## Requirements
### ADX-1 — resolve() and artist() accept a bare string (Axiom 0 first-try) [MUST] · S3-surface
- **Requirement:** Add a string overload to resolve(): `resolve(query: string | ResolveQuery, opts?)`. A bare string is inferred the same way artist() already detects: URL -> {url}; /^(discogs|mbid):/i -> strip scheme -> {discogs}/{mbid}; /^[0-9a-f]{64}$/i -> {cluster}; otherwise -> {q}. The ResolveQuery object union stays for explicit callers. The inference table is documented in JSDoc. resolve('') throws CrateValidationError{code:'exactly_one_of'}. Reuse the HEX64/LOCATOR regexes from §3.2 (single source). This closes the gap where crate.resolve('Four Tet') — an agent's most natural first call — is both a compile AND runtime error with no teaching path.
- **Acceptance:** resolve('Four Tet') sends ?q=Four+Tet; resolve('discogs:12345') sends the discogs id; resolve('https://...') sends ?url=; resolve(<64-hex>) sends cluster; the ResolveQuery object form still compiles and works; resolve('') throws CrateValidationError{code:'exactly_one_of'} whose .hint names the accepted forms.
- **SDD edit:** §3.1 resolve() — add the string overload + inference table; §3.2 reuse HEX64/LOCATOR

### ADX-2 — CrateError.toJSON() — structured errors survive JSON.stringify for log/handoff [MUST] · S2-errors
- **Requirement:** Add toJSON() to the abstract CrateError root returning a stable, documented envelope: { name, kind, code?, message, status?, retryable?, retryAfter?, requestId?, param?, hint?, docUrl?, next?, details? }. Subclasses extend via super.toJSON() spread (CrateTimeoutError adds timeoutMs, CrateValidationError adds param/hint, etc.). Deliberately EXCLUDE .raw (the ~2KB body escape hatch) and serialize .cause as {name,message} only — keep handoff payloads bounded and never leak response bodies/headers into logs. Export the envelope as a type `CrateErrorJSON`. Document in JSDoc: 'CrateError is JSON-safe — JSON.stringify(err) preserves kind/code/hint/next for logging and agent handoff.' This is dedupe of two HIGH findings (both lenses): a plain Error serializes to {} because name/message/stack are non-enumerable, so today every teaching field evaporates exactly at the agent's log/handoff step.
- **Acceptance:** For each concrete class, JSON.parse(JSON.stringify(err)) round-trips kind + class-specific fields non-empty (status/code/retryAfter for CrateAPIError, timeoutMs for CrateTimeoutError, param/hint for CrateValidationError); .raw and the raw .cause object are ABSENT from the serialized form; the toJSON envelope key set is contract-pinned (snapshot).
- **SDD edit:** §7 Error Model — add toJSON() to the abstract root + per-subclass extension + the CrateErrorJSON envelope type and its stability note

### ADX-3 — Exported, enumerable error-kind + error-code dictionary (Axiom 5) [MUST] · S2-errors
- **Requirement:** Export the taxonomy the SDD currently only proses. (1) `export const CRATE_ERROR_KINDS = ['api','network','timeout','abort','validation','parse','pagination','not_found'] as const; export type CrateErrorKind = typeof CRATE_ERROR_KINDS[number]` and type CrateError['kind'] against it. (2) Export `CrateErrorCode` literal union + frozen `CRATE_ERROR_CODES` covering server codes seen in the spec (rate_limited, payment_required, not_found, plus the generic api fallback) AND every SDK-minted client-side code (exactly_one_of, api_key_required, beacon_token_required, masters_arity, pagination_no_progress, pagination_malformed_page, node_fetch_missing, base_url_has_path, parse_error). Type .code as `CrateErrorCode | (string & {})` so agents get autocomplete on the known set without breaking on novel future server codes. (3) Export a documented `CRATE_ERROR_REGISTRY` record keyed by kind -> {retryable, clientSide, carries[], whenThrown} — generated from the §7 table, not re-prosed. Dedupe of two HIGH findings (one per lens). Without this an agent must read the SDD to learn the values it branches on.
- **Acceptance:** CRATE_ERROR_KINDS / CrateErrorKind / CrateErrorCode / CRATE_ERROR_CODES / CRATE_ERROR_REGISTRY all import from '@hosaka/crate'. A typecheck test asserts CrateErrorKind is the exact union of every concrete class's .kind literal (adding a class without registering its kind fails compile). A unit test asserts every client-side throw-site sets a .code present in CRATE_ERROR_CODES, and each CRATE_ERROR_REGISTRY entry's `retryable` matches isRetryable() for that kind's representative status.
- **SDD edit:** §7 Error Model — add the exported CRATE_ERROR_KINDS/CrateErrorKind, CrateErrorCode/CRATE_ERROR_CODES, CRATE_ERROR_REGISTRY; restate the §7 table as the spec for these objects

### ADX-4 — Client-side errors MUST author the exact fix (.hint/.next/.param) (Axiom 6) [MUST] · S2-errors
- **Requirement:** Make .hint + .next + .param (where applicable) REQUIRED non-empty on every CLIENT-side throw — independent of the server, since the SDK authored the message and knows the fix. The §7 defensive-read rule stays ONLY for fields sourced from the open server Error schema. Concrete content per throw-site: key-gated-without-apiKey -> code:'api_key_required', param:'apiKey', hint:'<method>() requires an API key', next:'new Crate({ apiKey: process.env.CRATE_API_KEY })'; resolve arity!==1 / empty string -> code:'exactly_one_of', hint:'pass exactly one of url|q|cluster|discogs|mbid', next:'crate.resolve({ q: "Four Tet" })'; missing beaconToken -> code:'beacon_token_required', param:'beaconToken' + corrected opts shape; masters() over-limit -> code:'masters_arity', param:'ids', hint naming the 1..100 bound + 'chunk into batches of <=100', next with a chunking call (the actual count in the message); baseUrl-with-path -> code:'base_url_has_path', next:'new Crate({ baseUrl: "https://crate.0xhoneyjar.xyz" })'; Node<18 fetch guard -> code:'node_fetch_missing', hint:'pass opts.fetch or upgrade to Node 18+'. Also: CrateNotFoundError from artist() locator-miss sets .hint to the resolve note verbatim and .next:'use artistOrNull() to get null instead of a throw'. Provide a default docUrl base for client-side kinds. Dedupe of MEDIUM (rubric) + HIGH (Lens B) + the LOW not-found steer.
- **Acceptance:** A test enumerates EVERY client-side throw-site and asserts the thrown CrateError has non-empty .hint and .next (and .param where applicable), .next is a copy-pasteable corrected call (contains the method name + a valid argument form), and no client-side CrateError has empty .hint. The locator-miss CrateNotFoundError carries the resolve note in .hint and 'artistOrNull' in .next.
- **SDD edit:** §7 (split the defensive-read rule: server-sourced fields defensive, client-side fields mandatory) + §3/§3.1/§3.4/§3.5 throw-sites (specify the exact hint/next/param) + §3.2 not-found steer

### ADX-5 — JSDoc-with-@example + @throws on every public method (the type surface IS the agent's docs) [MUST] · S3-docs
- **Requirement:** Mandate a JSDoc block on every public method, the Crate constructor, CrateOptions/RequestOptions fields, and every exported error guard, containing: (a) one-line summary, (b) @param for non-obvious args, (c) at least one @example showing a realistic call, (d) @throws naming the CrateError kinds it can throw (cross-referencing §7), (e) @see linking related methods (resolve<->artist, bulk<->bulkAll<->index, artist<->artistOrNull). Forgiving-input methods (resolve, artist) MUST show one @example per accepted input shape (incl. the bare-string and numeric-id cases from ADX-1/ADX-9). Examples are anon-safe (never call key-gated methods in anon examples). Enforce with eslint-plugin-jsdoc (require @example + @throws on exported members) OR a lightweight AST contract test. Dedupe of two findings (HIGH rubric + MEDIUM Lens B). For a TS SDK the .d.ts + hover docstrings are the agent's primary documentation; the SDD enforces type coverage rigorously but specifies zero doc coverage.
- **Acceptance:** A lint/AST gate in CI fails if any exported public method/guard lacks a JSDoc @example or a @throws tag; spot-check asserts resolve()/artist()/bandcamp.bulkAll()/search()/master() each have an @example (one per accepted input variant for resolve/artist) and a @throws.
- **SDD edit:** §9 Testing — add a doc-coverage gate; §3 surface — note every public member carries JSDoc+@example+@throws

### ADX-6 — README 'Using from an AI agent' recipe (the discoverability surface) [MUST] · S3-docs
- **Requirement:** Mandate a README section 'Using from an AI agent' (beyond the existing MIT/ToS notice) containing copy-pasteable, anon-safe, type-checked snippets: (1) zero-config anonymous quickstart — new Crate() then await crate.artist('Four Tet'); (2) the canonical error-recovery pattern — try/catch -> switch(err.kind), branch on err.code, read err.hint/err.next, use the exported guards (not instanceof), and JSON.stringify(err) is safe for handoff; (3) the bandcamp.bulkAll() for-await + .pages() pagination recipe incl. resume-from-cursor (ADX-8); (4) the key-gated-vs-anonymous method distinction and the exact CrateValidationError thrown without a key; (5) pointers to crate.index() (live root map) and CRATE_ERROR_CODES/CRATE_ERROR_REGISTRY/VERSION/CRATE_RESOURCES (static self-description). Make this an explicit acceptance criterion of the S3 README task — not just the license notice. Dedupe of two findings (HIGH rubric + MEDIUM Lens B).
- **Acceptance:** README has a top-level 'Using from an AI agent' section; its quickstart references only anon methods, demonstrates switch(err.kind)+err.code+err.next, shows JSON.stringify(err), shows bulkAll iteration, and references crate.index() + the exported registries. A CI doctest/example-typecheck step compiles the snippet under tsc --noEmit so it can't rot.
- **SDD edit:** §8 Build & Packaging (README mandate) + sprint Task 3.9 acceptance criteria

### ADX-7 — SDK self-description surface: VERSION, CRATE_RESOURCES, index() framed as discovery entrypoint (Axiom 9) [SHOULD] · S3-surface
- **Requirement:** Export the client's own capability map so an agent can self-orient without reading source. (1) Export `VERSION` (string, build-stamped from package.json via tsup `define`, NOT a runtime require — keeps zero-dep + tree-shakeable). (2) Export a frozen `CRATE_RESOURCES` metadata map: one entry per public method with { method, endpoint, returns (type name), auth: 'anon'|'key'|'beacon', retryable, idempotent } — single-sourced from the §3 surface table. (3) Re-export CRATE_ERROR_CODES/CrateErrorKind (from ADX-3) as part of the robot-docs root. (4) In index()'s JSDoc, frame it explicitly as the agent's live discovery entrypoint and @see it from the Crate class doc. Dedupe of two MEDIUM findings. Keep MEDIUM thickness: this is exported constants derived from the §3 table, NOT a runtime introspection engine; a static Crate.describe() returning {version,resources,errorCodes} is OPTIONAL/nice.
- **Acceptance:** import { VERSION, CRATE_RESOURCES } from '@hosaka/crate' resolves; a test asserts VERSION === package.json version and that CRATE_RESOURCES has one entry per public method in §3 with auth tiers matching the table (anon/key/beacon counts); index()'s JSDoc has an @example and is @see-referenced from the Crate class docstring.
- **SDD edit:** §3 surface map (note CRATE_RESOURCES single-sourced from the table; index() framed as discovery entrypoint) + §8 (VERSION build-stamp via tsup define)

### ADX-8 — Resumable pagination: expose cursor handle + treat maxPages as a clean stop, not an error [SHOULD] · S3-surface
- **Requirement:** Two minimal changes to §6, surfacing state the generator already holds (no new machinery). (1) Add a `.cursor` getter on the dual-iterable handle (last next_cursor consumed, null at end) and make CratePaginationError carry a documented public `.lastCursor` re-passable into bandcamp.bulk({cursor}). Document the resume recipe in JSDoc/README: stop/catch -> read cursor -> bulk({cursor}) to continue (no re-fetch). (2) Distinguish the two pagination outcomes: hitting `maxPages` is a DELIBERATE user cap, not a fault — it MUST end iteration cleanly with an observable `truncated:true` + lastCursor signal on the handle, NOT a thrown CratePaginationError. Only a genuinely non-advancing/cycling cursor throws CratePaginationError{code:'pagination_no_progress'}; a malformed page throws {code:'pagination_malformed_page'}. Dedupe of the MEDIUM resumability finding + the MEDIUM masters/maxPages dead-end finding (pagination half).
- **Acceptance:** A test paginates two pages, stops via maxPages:1, reads handle.cursor, passes it to bulk({cursor}) and asserts page 2 is fetched with no re-fetch of page 1; hitting maxPages ends iteration with truncated:true (NO thrown error); a cursor-cycle case throws CratePaginationError{code:'pagination_no_progress'} and its .lastCursor is re-passable to bulk().
- **SDD edit:** §6 Pagination — add .cursor getter + .lastCursor public field + truncated-vs-thrown distinction with distinct codes

- **Acceptance:** A mocked test pins artist('Four Tet') for both the resolves-natively and sparse-identity cases (if fallback chosen, the resolve({q}) second hop fires only on sparse identity); artist('1234567') is pinned to the chosen path (resolve({discogs}) hop OR CrateValidationError whose .next names discogs:); the §13 'DX confirm pending' line is replaced by a documented decision before S3 closes.

## Rejected as over-engineering (kept MEDIUM)
- Special agent-only API mode / a separate 'agent client' surface — rejected: the brief explicitly bars agent-mode bloat. The same typed surface + structured errors serve humans and agents; forking the API doubles the maintenance and contract-test burden for no DX gain.
- Telemetry / usage-beacon auto-emission of error events for 'agent observability' — rejected: out of scope, privacy-hostile for a zero-dep client, and the SDK already never logs keys (§11). Agents log their own errors via toJSON (ADX-2).
- A runtime introspection/reflection engine that walks the live class hierarchy to build CRATE_RESOURCES — rejected as over-engineering: CRATE_RESOURCES is single-sourced as a frozen literal from the §3 table (ADX-7), which is simpler, tree-shakeable, and equally machine-readable. Reflection adds code for no agent benefit.
- Mandatory Crate.describe() method — downgraded to OPTIONAL inside ADX-7: the exported VERSION/CRATE_RESOURCES/CRATE_ERROR_CODES constants already give agents a JSON-serializable self-description; a wrapper method is sugar, not a requirement, and would need its own test/contract surface.
- Auto-fallback/retry across DIFFERENT resolve interpretations (e.g. try {q} then {cluster} then {discogs} on miss) — rejected: violates Axiom 12 determinism and silently masks honest gaps; the SDK surfaces resolved_via verbatim and lets the agent decide. ADX-1 only does syntactic single-classification of a bare string, no speculative multi-try.
- Embedding full prose docs / a generated docs site in the package — rejected as scope creep beyond MEDIUM; JSDoc @example on the type surface (ADX-5) + the README agent recipe (ADX-6) are the right-sized self-documentation for a library. A docs site is a separate post-V1 effort.
- Making .raw or .cause part of toJSON for 'full fidelity' — rejected: deliberately excluded in ADX-2 to keep handoff payloads bounded (~2KB body) and avoid leaking response bodies/request context into agent logs; the escape hatch stays available on the live error object for the rare case that needs it.
- Auto-issuing/auto-wiring the beacon JWT to make searchEvents.* zero-config — rejected: the public contract genuinely does not expose token issuance (§3.4, §13 flag #1); inventing a flow would be guessing. Caller-supplied beaconToken with a teaching CrateValidationError (ADX-4) is the honest design until the upstream gap closes.
