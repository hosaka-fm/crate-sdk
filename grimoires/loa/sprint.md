## Executive Summary

V1 of `@hosaka/crate` is a single, greenfield TypeScript package: a thin, typed, **zero-runtime-dependency** client over crate's public REST contract, in the `stripe-node` / `@supabase/supabase-js` "medium thickness" sweet spot. Per the SDD's full-surface scope (§13, jani override), V1 wraps the **entire** public surface — conveniences, anonymous reads, key-gated reads, and beacon endpoints — built/tested/tagged in-repo with **publish gated OFF** until crate's API key model lands.

The work decomposes cleanly along the SDD's module dependency order: generated types and build scaffold first (everything compiles against them), then the shared transport engine (`retry` → `errors` → `http`, the load-bearing funnel every method uses), then the public surface (client, conveniences, resources, re-exports) capped with end-to-end goal validation.

**Sprint Duration:** 2.5 days each
**Estimated Completion:** 2026-07-04 (≈7.5 working days)

## Sprint 2: Transport core — retry, errors, http funnel

**Scope:** MEDIUM (6 tasks)
**Duration:** 2.5 days
**Dates:** 2026-06-29 — 2026-07-01

### Sprint Goal
Build and fully unit-test the single shared transport engine — pure retry math, the branded `CrateError` hierarchy, and the `http.ts` funnel every public method delegates to — so the surface layer in Sprint 3 is a thin set of typed URL builders over a proven core.

### Deliverables
- [ ] `src/retry.ts` — pure `isRetryable()`, `computeDelay()` (full-jitter), `parseRetryAfter()` (delta-seconds + HTTP-date), no I/O
- [ ] `src/errors.ts` — abstract `CrateError` root with `kind` discriminant, all 8 concrete classes, `Symbol.for` brand, all brand-based type guards
- [ ] `src/http.ts` — URL build, header precedence, Node-18 guard, retry loop, per-attempt timeout/abort, body parse, 2xx/non-2xx dispatch
- [ ] Transport unit-test suite (retry, backoff math, error mapping, timeout vs abort, parse failure) using `undici` `MockAgent` + fake timers

### Acceptance Criteria
- [ ] **Retry**: 429/500/503/504 retried with full-jitter backoff; Retry-After precedence = header (delta **and** HTTP-date) > body `retry_after_seconds` > jittered backoff; clamped to `maxRetryAfterMs` and remaining `totalDeadlineMs` (sdd.md:155)
- [ ] **No-retry**: 400/401/402/404/413 and `AbortError` are never retried; beacon POSTs never retried (sdd.md:153–154)
- [ ] **Idempotency** flag set per-request at the method layer: GETs + read-shaped POSTs (`masters/batch`, `wayfind/*`) retryable, beacons false (sdd.md:154)
- [ ] **Errors**: each concrete class sets `kind` + hardcoded `this.name` literal; all guards are **brand-based** and survive ESM/CJS duplication; `CrateAPIError` carries `.code/.status/.retryable/.details/.retryAfter/.requestId/.raw` and reads teaching fields (`.hint/.docUrl/.next/.param`) **defensively** (sdd.md:184, 188)
- [ ] **429 bodies** parsed as `RateLimited` (`error:'rate_limited'`, required `retry_after_seconds`); message composition never trusts `Response.statusText` (status→reason table, double-space collapse) (sdd.md:184, 190)
- [ ] **Timeout/abort**: per-attempt fresh `AbortController` + `setTimeout` cleared in `finally` (not `AbortSignal.timeout`); caller `signal` composed; timeout vs caller-abort disambiguated via `signal.reason` → `CrateTimeoutError` vs `CrateAbortError`; listeners removed on settle (sdd.md:158)
- [ ] **Parse**: malformed JSON (2xx **or** error) → `CrateParseError` carrying `.status`/`.raw`; 204 → `undefined`; Node-18 missing-`fetch` guard throws a teaching `CrateError` (sdd.md:144, 146)
- [ ] Pure retry functions unit-tested in isolation; `vi.useFakeTimers` + `advanceTimersByTimeAsync` drive backoff timing deterministically
- [ ] **Agent ergonomics (sdd.md §15)**: `CrateError.toJSON()` returns the stable `CrateErrorJSON` envelope, excludes `.raw`/raw `.cause`, round-trips through `JSON.stringify` (ADX-2); `CRATE_ERROR_KINDS`/`CrateErrorKind`/`CrateErrorCode`/`CRATE_ERROR_CODES`/`CRATE_ERROR_REGISTRY` exported, with a typecheck test asserting `CrateErrorKind` == exact union of class `.kind` literals (ADX-3); **every client-side throw carries non-empty `.hint`+`.next`(+`.param`)** with copy-pasteable corrected calls — defensive-read rule applies only to server-sourced fields (ADX-4)

### Security Considerations
- **Trust boundaries**: HTTP response bodies are **untrusted** — every body goes through guarded `JSON.parse`; teaching fields read defensively (never assumed present); page/error shapes validated before use. `.raw` body escape hatch capped (~2KB) and **never** includes request headers (sdd.md:215).
