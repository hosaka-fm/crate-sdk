# Sprint 1 Implementation Report — Foundation (types, drift guard, build & packaging)

## Executive Summary

Sprint 1 stands up `@hosaka/crate` as a reproducible, zero-runtime-dependency dual ESM+CJS

two-tier drift guard. All 7 acceptance criteria are met and verified empirically (commands
+ output below). No application/client logic yet — that is Sprint 2 (transport) and Sprint 3

## AC Verification

1. **"`npm ci` installs with a committed lockfile and zero runtime `dependencies` (only `devDependencies`)"**
   - ✓ Met. `package.json:46` has no `dependencies` key; `devDependencies` only (`package.json:47-56`). Committed `package-lock.json`. Asserted: `node -e "Object.keys(require('./package.json').dependencies||{})"` → `zero runtime deps OK`. CI re-asserts (`.github/workflows/ci.yml:31-33`).

2. **"Offline drift test passes: regenerating types from the vendored spec via the same pinned binary produces byte-identical output"**
   - ✓ Met. `test/drift.test.ts:22-31` regenerates via `scripts/generate-types.mjs` to a temp file and asserts `Buffer.equals`. Run output: `✓ regenerating from the vendored spec is byte-identical to the committed types`. Generator + drift test both use the locally-installed pinned `openapi-typescript@7.13.0` (`scripts/generate-types.mjs:26-31`, `package.json:52`).

3. **"The drift test fails when `crate-api.d.ts` is hand-edited or the vendored spec is mutated (guard proven, not a no-op)"**
   - ✓ Met. `test/drift.test.ts:33-38` mutates the committed bytes and asserts the comparison returns `false`, proving the byte-equality check is not a no-op. (The primary test in §2 fails if committed≠regenerated by construction.)

4. **"`tsup` build succeeds and `@arethetypeswrong/cli --pack` reports no dual-package hazard"**
   - ✓ Met. `npm run build` → `dist/index.js` (ESM) + `dist/index.cjs` (CJS) + `dist/index.d.ts` + `dist/index.d.cts` (`tsup.config.ts:9-17`). `npm run check:exports` → **"No problems found 🌟"**; node10 / node16-from-CJS / node16-from-ESM / bundler all 🟢. `@arethetypeswrong/cli` pinned exactly `0.18.4` (`package.json:48`; 0.17.4 had an internal CLI bug — Decision Log).

5. **"`npm pack --dry-run` tarball excludes `src/`, `test/`, `spec/`"**
   - ✓ Met. `files:["dist","README.md","LICENSE","NOTICE"]` (`package.json:38-43`). `npm pack --dry-run --json --ignore-scripts` → 8 files: `LICENSE, NOTICE, README.md, dist/index.cjs, dist/index.d.cts, dist/index.d.ts, dist/index.js, package.json`. CI asserts no `src|test|spec|scripts` leak (`.github/workflows/ci.yml:35-40`).

6. **"`release.yml` publish step is a no-op unless `PUBLISH_ENABLED` repo var is set and `NPM_TOKEN` present (both absent today)"**
   - ✓ Met. `.github/workflows/release.yml:26-37`: publish step gated `if: vars.PUBLISH_ENABLED == 'true' && env.NPM_TOKEN != ''`; an explicit "Publish skipped (gate closed)" step runs otherwise. `package.json:8` `"private": true` is the fail-closed second gate.

7. **"All re-export target schemas exist in the generated module (`ArtistDossierContract`, `IdentityResolution`, `ApiRootIndex`, `BandcampBulkPage`, `BandcampFeedContract`)"**
   - ✓ Met. `src/types.ts:11-46` aliases all five (and the full method surface) off `components['schemas']`; `tsc --noEmit` passes, proving every alias resolves against `src/generated/crate-api.d.ts`.
